## §1. 日常水質タブの 2サブタブ化

### 1.1 構成

加計の日常水質タブを **2サブタブに分割** する：

```
日常水質（メインタブ）
├─ 上段使用量（サブタブ1）   … 11項目／週、月-金+翌週月の6セル
└─ 下段水質測定（サブタブ2） … 34項目／日 × 5日
```

サブタブ切替UIは加計の機械設備サブタブと同じパターンを流用（ボタン群＋表示切替）。

### 1.2 上段使用量の項目（11項目）

加計の上段に **余剰汚泥流量 No.1, No.2 を追加** して 11項目にする。

| # | 項目名 | 単位 | 加計差分 |
|---|---|---|---|
| 1 | 全日電力量 | Kwh | |
| 2 | 力測電力量 | Kwh | |
| 3 | 無効電力量 | Kvar | |
| 4 | デマンド | Kw | |
| 5 | 水道使用量 | ㎥ | |
| 6 | 軽油（残量） | ℓ | |
| 7 | 返送汚泥流量（№1） | ㎥ | |
| 8 | 返送汚泥流量（№2） | ㎥ | |
| 9 | 放流流量 | ㎥ | |
| 10 | **余剰汚泥流量（No.1）** | ㎥ | **新規** |
| 11 | **余剰汚泥流量（No.2）** | ㎥ | **新規** |

各項目: 月〜金（5日）+ 翌週月曜（差分計算用）の **6セル/項目** 入力。

### 1.3 下段水質測定の項目（34項目/日）

加計の下段を以下のように拡張：

- **2系統対応**：水温・外観・臭気・PH・SV・MLSS・汚泥界面 を ディッチ1/ディッチ2 の2系統分用意
- **SV を SV30/SV24h × 1系/2系 の4項目に簡素化**（加計の SV10/SV20 削除、2系統分追加）
- **終沈3項目を新規追加**：水温・透視度・PH の各 No.1終沈/No.2終沈 = 計6セル

完全な項目リストは元指示書 §5.1.2 を参照。

### 1.4 GAS 書き込み先（日常水質シート）

書き込み先のセル位置は元指示書 §8.1〜§8.2 とテンプレ `筒賀_テンプレート_v5.xlsx` を参照。

各週ブロック開始行: `r1, r53, r105, r157, r209`（オフセット 0, 52, 104, 156, 208）

ブロック内の項目行（オフセットを足して使う）:
- r4-r12: 使用量9項目（旧）+ 余剰汚泥流量はここではなく日常電気設備に書き込み
- r16-r19: 水温（流入/D1/D2/放流）
- r24-r25: 透視度（流入水/放流水）
- r30-r33: PH（流入/D1/D2/放流）
- r34-r37: SV1系SV30/SV1系24h/SV2系SV30/SV2系24h
- r38-r39: MLSS（1系/2系）
- r40-r41: 汚泥界面（1系終沈/2系終沈）
- r42: 残留塩素
- r43: 塩素投入量

列: I=月, J=火, K=水, L=木, M=金, N=翌週月（使用量のみ）

## §4. GAS の複合転記ルール（仕様 8.2）

### 4.1 全体方針

アプリで入力されたデータを GAS が **複数のシートに転記** する。Excel側の数式参照は使わない（月またぎで無効になるため）。

GAS の `exportExcel`（または `writeData`）関数で、以下の転記ロジックを **すべて実装** する。

### 4.2 終沈3項目（アプリ入力 → 水質管理報告）

| アプリ入力 | 水質管理報告の書き込み先 |
|---|---|
| 終沈水温 No.1 | F列前半（r9〜r38 の平日のみ）|
| 終沈水温 No.2 | G列前半 |
| 終沈透視度 No.1 | P列前半 |
| 終沈透視度 No.2 | Q列前半 |
| 終沈PH No.1 | N列後半（r50〜r79 の平日のみ）|
| 終沈PH No.2 | O列後半 |

平日マッピング（3月の例）:
- r9/r50=3/2(月), r10/r51=3/3(火), ... r38/r79=3/31(火)

### 4.3 重複項目（アプリ入力 → 日常水質 + 水質管理報告）

水質管理報告にも書き込みが必要な項目：

| アプリ入力 | 日常水質 | 水質管理報告 |
|---|---|---|
| 水温（流入） | r16 | C列前半 |
| 水温（D1） | r17 | D列前半 |
| 水温（D2） | r18 | E列前半 |
| 水温（放流） | r19 | H列前半 |
| 透視度（流入水） | r24 | O列前半 |
| 透視度（放流水） | r25 | R列前半 |
| PH（流入） | r30 | K列後半 |
| PH（D1） | r31 | L列後半 |
| PH（D2） | r32 | M列後半 |
| PH（放流） | r33 | P列後半 |
| 残留塩素 | r42 | Q列後半 |
| SV30（1系） | r34 | R列後半 |
| SV30（2系） | r36 | S列後半 |

### 4.4 余剰汚泥流量（アプリ入力 → 日常電気設備）

日常水質シートには書き込まず、**日常電気設備シートの余剰汚泥流量計セル** に書き込む：

| アプリ入力 | 日常電気設備の書き込み先（G-K列、月-金）|
|---|---|
| 余剰汚泥流量 No.1 | r56（1週）/ r157（2週）/ r257（3週）/ r357（4週）/ r457（5週）|
| 余剰汚泥流量 No.2 | r57 / r158 / r258 / r358 / r458 |

### 4.5 燃料タンク残量（アプリ入力（軽油残量）→ 日常電気設備）

アプリで入力する「軽油（残量）」を、日常水質シートと **日常電気設備の燃料タンク残量セル** の両方に書き込む：

電気設備の点検項目として入力された DO値 を、両シートに書き込む：

## §5. 実装順序の推奨

協調変更のリスクを下げるため、以下の順序を推奨：

1. **GAS の転記ロジックを先に実装**（§4 全部）
   - HTML/JS は後回しでも、テストデータをスプレッドシートに直接入れて GAS の出力を検証できる
   - 転記マッピングが正しいかを早期に確認

2. **日常水質タブの 2サブタブ化**（§1）
   - 上段使用量（11項目）から実装
   - 下段水質測定（34項目）の終沈3項目追加に注意

3. **日常機械設備の 3サブタブ化**（§2）

4. **日常電気設備の 3サブタブ化**（§3）
   - DO値入力UIに注意（GAS転記との連動）
   - 余剰汚泥流量入力UIに注意（日常水質タブの上段から転記される、入力UIは点検表2/3にあり）

5. **統合テスト**
   - 全タブで入力 → 保存 → Excel出力 → テンプレに正しく書き込まれているか確認
